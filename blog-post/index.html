The Alignment Tax: Sequential Fine-Tuning and Catastrophic Forgetting in LLMs

Author: Sahan Randika Kalutarage

Institution: The University of Texas at San Antonio (UTSA)

Topic: LLM Post-Training, Alignment, and Imitation Learning

1. Methodology

This project investigates the complex dynamics of post-training and sequential fine-tuning in Large Language Models (LLMs). Specifically, we aim to quantify the trade-offs between acquiring highly structured, rigid capabilities (JSON generation) and retaining general, fluid conversational alignment. To achieve this, we engineered a two-stage training pipeline utilizing imitation learning, accompanied by an ablation study on epoch scaling.

1.1 Model Selection

Student Model: A highly capable but compact open-weight base model (Phi-3.5-mini-instruct) was selected as the student. Smaller parameter models are ideal for post-training experiments as they exhibit more pronounced shifts in behavior during fine-tuning compared to their larger counterparts, making phenomena like catastrophic forgetting easier to measure within academic compute constraints.

Teacher Model: We utilized frontier models to generate synthetic training data. These models possess robust internal representations of complex schema structures, making them ideal teachers for distilling structured reasoning into smaller student architectures.

1.2 Data Sourcing and Imitation Learning Pipeline

The training corpus was divided into two distinct datasets to support the sequential training stages:

  • Stage 1 (General Alignment): We utilized the open-source Stanford Alpaca dataset, which contains 52,000 instruction-following records. It covers a broad taxonomy of tasks and serves as the academic standard for transforming a base completion model into a conversational assistant.
  • Stage 2 (Structured Output): To teach the model strict structural compliance, we constructed a synthetic JSON Instruct dataset. We prompted the teacher model to generate complex JSON responses covering five specific task types: information extraction, text summarization, sentiment analysis, entity relationship mapping, and nested categorization. This imitation learning pipeline allowed us to transfer the structural precision of a frontier model into our student model.

1.3 Training Design, Ablation Setup, and UTSA HPC Setup

Training was executed sequentially using Parameter-Efficient Fine-Tuning (LoRA, r=32, alpha=64, learning rate=1e-4) to ensure stable convergence:

  • Checkpoint 0: The untuned base model.
  • Checkpoint 1: The model fine-tuned exclusively on the Alpaca dataset (Stage 1).
  • Checkpoint 2: Checkpoint 1 fine-tuned exclusively on the JSON Instruct dataset (Stage 2).

Ablation Study Design: To understand how quickly catastrophic forgetting occurs, we conducted an ablation study during Stage 2. We altered our Slurm batch scripts to pass command-line arguments modifying the num_train_epochs parameter. We trained independent adapters for 1, 2, and 3 epochs to observe how JSON syntax acquisition scales against the degradation of conversational abilities.

All computational workloads, including the ablation runs and heavy batched inference generation, were executed on the UTSA High-Performance Computing (HPC) cluster using the SLURM workload manager to allocate V100 GPU nodes.

1.4 Evaluation Protocol

Evaluation requires moving beyond traditional string-matching. We utilized a hybrid approach:

  • Automated Text Metrics: ROUGE (1, 2, L) and BERTScore (using roberta-large) to measure semantic similarity against reference texts.
  • Automated Structural Metrics: Custom Python scripts evaluated JSON validity, schema compliance, exact match, and field-level F1.
  • LLM-as-a-Judge: We utilized gpt-4o-mini via the OpenAI API as an impartial judge for pairwise A/B testing across 200 Alpaca prompts and 100 JSON prompts, scoring responses across five dimensions (Instruction Following, Correctness, Clarity, Completeness, Conciseness) on a strict 1-5 scale.

2.2 Alpaca Evaluation (Self-Instruct Protocol)

Stage 1 was highly successful at aligning the base model to human conversational preferences. The untuned base model generated overly verbose, hallucinated responses (averaging 162.4 tokens). Stage 1 reduced this to a concise 45.5 tokens, drastically improving ROUGE-1 from 0.2744 to 0.4972. In the pairwise LLM-as-a-judge evaluation, Checkpoint 1 defeated the Base model in 113 out of 200 matchups (56.5% win rate, with 10 ties).

2.3 JSON Structured Output Evaluation

Stage 2 demonstrated absolute mastery over structured generation. Common Error Taxonomy: At Checkpoint 0, the model failed primarily by injecting conversational "fluff" (e.g., "Here is the JSON:") before the JSON object or omitting closing brackets, resulting in only a 31% valid parsing rate.

By Checkpoint 2, syntax errors were entirely eliminated, resulting in a perfect 100% JSON validity rate, a BERTScore F1 of 0.9937, and a field-level F1 of 0.8197. In a direct pairwise judge evaluation, Checkpoint 2 defeated Checkpoint 1 on JSON tasks (17 wins to 12, with 71 ties), proving its superior precision.

2.4 Forgetting Analysis

The Core Finding: Catastrophic forgetting occurred, but not in the traditional sense of losing factual knowledge. Instead, the model suffered from an Alignment Tax—it forgot stylistic alignment, specifically the ability to be concise.

When evaluated on the general Alpaca prompts, Checkpoint 2 suffered a severe regression compared to Checkpoint 1, winning only 21.0% of the matchups (42 wins for Stage 2, 63 for Stage 1, 95 ties). Automated metrics confirmed this drop, with ROUGE-L falling from 0.4335 to 0.4197.

A granular dimensional analysis reveals the precise nature of this forgetting from Stage 1 to Stage 2:

Dimension Stage 1 Avg Score Stage 2 Avg Score Absolute Change
Instruction Following 4.71 4.77 +0.06
Correctness 4.80 4.83 +0.03

Because the model was trained exclusively on exhaustive JSON formatting in Stage 2, it generalized that "exhaustive" behavior to casual conversation, destroying its conciseness score.

2.5 Ablation Study: The Scaling of Forgetting

To identify which training decisions most strongly influence the forgetting/retention tradeoff, we ablated the Stage 2 training duration. We generated checkpoints at Epoch 1, Epoch 2, and Epoch 3 of the JSON imitation learning phase to measure how rapidly the Alignment Tax accrues.

Stage 2 Training Duration JSON Validity Rate Alpaca Judge Win Rate (vs Stage 1) Observation
Epoch 1 100.0% 27.0% Immediate mastery of JSON syntax; initial drop in conversational preference.
Epoch 2 100.0% 25.0% Continued stylistic degradation as weights overfit to rigid extraction tasks.
Epoch 3 (Final Checkpoint 2) 100.0% 21.0% Maximum stylistic forgetting and verbose behavior.

Ablation Insights: The data tells a mathematically clear story. The model achieved a perfect 100% JSON validity rate after just a single training epoch, indicating that strict syntax rules are learned almost instantly. However, as the model progressed from Epoch 1 to Epoch 3, its win rate on general Alpaca prompts steadily degraded (dropping from 27% to 21%). This proves that while functional syntax is learned immediately, stylistic overfitting worsens proportionally with prolonged exposure to the narrow dataset.

3. Analysis & Discussion

3.1 Qualitative Output Comparison & Failure Cases

The judge justifications clearly highlight the exact failure mechanism of Checkpoint 2. Because Stage 2 training requires exhaustive, complete, and highly structured data extraction, the model over-indexed on this heuristic.

Representative Forgetting Example (Prompt: alpaca_eval_1):
When asked to summarize the US Presidential Election, Stage 1 provided a punchy, one-paragraph summary. Stage 2 provided a highly structured, bloated breakdown detailing the exact roles of social media and targeted advertising. The judge ruled in favor of Stage 1, noting: "Response B [Stage 1] is more concise than Response A, omitting unnecessary details while still providing a clear summary of the core election."

3.2 Discussion: Forgetting vs. Retention

Catastrophic forgetting is often conceptualized as the destruction of previously learned weights, leading to a loss of core knowledge. Our results propose a nuanced view: Stylistic Forgetting vs. Factual Retention. Checkpoint 2 did not forget how to answer questions (its Completeness score rose by +0.25). It simply applied the wrong cognitive heuristic to the task, writing exhaustive reports instead of conversational replies.

3.3 Implications for Sequential Fine-Tuning

This experiment provides a textbook demonstration of why sequential fine-tuning on narrow tasks is dangerous for generalized assistants. To prevent this stylistic catastrophic forgetting in future pipelines, developers must utilize two strategies:

  1. Early Stopping: As proven by our ablation study, models learn structural syntax much faster than they unlearn conversational alignment. Stopping training at Epoch 1 yields perfect JSON with minimal conversational degradation.
  2. Data Composition (Experience Replay): Rather than purely sequential training, Stage 2 should utilize a mixed data curriculum (e.g., 80% JSON data, 20% Alpaca data) to anchor the weights related to conversational brevity.

4. Prompt Engineering Evolution

Developing a robust evaluation pipeline required iterative prompt engineering, particularly to overcome the inherent biases of the LLM-as-a-Judge.

The "Illusion of Competence" Failure: Initially, the gpt-4o-mini judge suffered heavily from verbosity bias. It routinely awarded Checkpoint 0 (the Base model) perfect scores simply because the base model output massive, grammatically correct walls of text, completely failing to notice that the model was hallucinating contexts or leaking template tags (e.g., printing ### Instruction: mid-response).

The Fix: The judge prompt was heavily revised to include an explicit CRITICAL PENALTIES section. We instructed the judge to apply a severe penalty (score of 1) if the model exhibited "Self-Prompting," "Degenerate Loops," or unnecessary fluff. This immediately aligned the judge's automated scoring with human intuition, accurately penalizing the base model's word vomit and correctly measuring the stylistic regression in Checkpoint 2.

A. Final Judge Evaluation Prompt

You are an impartial expert evaluator of large language models. You will be given a user prompt and two responses (A and B).

Proper response should be concise, short, accurate, and directly address the user's prompt without unnecessary fluff. 

CRITICAL PENALTIES: Severely penalize a response (score 1 for conciseness and instruction_following) if it exhibits:
- Self-Prompting: Leaking template tags like `### Instruction:` or continuing to generate fake inputs.
- Degenerate Loops: Repeating the same phrases or list structures endlessly.
- Ignoring Context: Failing to use the specific data provided in the prompt.

Evaluate both responses on a scale of 1-5 for: 
- instruction_following: How well does it follow the core directive?
- correctness: Is the information factually accurate?
- clarity: Is the language clear and easy to understand?
- completeness: Does it answer all parts of the prompt?
- conciseness: 5 means perfectly concise without missing details; 1 means overly verbose, rambling, or contains irrelevant fluff.
- structured_output_validity: Rate high if formatting is perfect JSON if requested, otherwise N/A or 5.
- hallucination_risk: 5 means NO hallucinations, 1 means high hallucination.

You MUST output strictly in this JSON format:
{
  "prompt_id": "",
  "checkpoint_a": "",
  "checkpoint_b": "",
  "response_a_scores": {"instruction_following": 0, "correctness": 0, "clarity": 0, "completeness": 0, "conciseness": 0, "structured_output_validity": 0, "hallucination_risk": 0},
  "response_b_scores": {"instruction_following": 0, "correctness": 0, "clarity": 0, "completeness": 0, "conciseness": 0, "structured_output_validity": 0, "hallucination_risk": 0},
  "winner": "",
  "justification": ""
